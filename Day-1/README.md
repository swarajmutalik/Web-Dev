# Day-1 Overview - 

## How Websites Work

Websites operate on a client-server model. When a user requests a webpage, their browser acts as the client and sends a request to a server. The server processes this request and sends back a response containing the requested webpage.

## Key Components

- Frontend: HTML, CSS, and JavaScript handle the structure, style, and interactivity of the webpage.
- Backend: Server-side languages like Python, Node.js, or Ruby manage application logic and interact with databases.
- Database: Data is stored and retrieved dynamically from databases like MySQL, PostgreSQL, or MongoDB.
- HTTP Protocol: Communication between clients and servers relies on the Hypertext Transfer Protocol (HTTP).
- DNS: The Domain Name System translates domain names into IP addresses.

## Technology Stack for a Website
The technology stack for a website includes: - 

### Frontend Technologies

- **HTML, CSS, JavaScript**: Fundamental languages for webpage structure, styling, and interactivity.

### Frontend Frameworks/Libraries

- **React.js, Angular, Vue.js**: Popular libraries/frameworks for building dynamic user interfaces.

### Backend Technologies

- **Node.js, Python**: Backend languages for server-side logic.

### Backend Frameworks

- **Express.js, Django**: Frameworks facilitating backend development.

### Database Technologies

- **MySQL, PostgreSQL, MongoDB**: Commonly used databases for storing and retrieving data.

### Web Servers

- **Nginx, Apache**: Web servers responsible for serving web content.

### Version Control

- **Git**: Essential for tracking changes in source code during development.

### Deployment Tools

- **Docker, Heroku**: Tools for packaging and deploying applications.
