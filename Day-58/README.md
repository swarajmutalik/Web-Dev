## Introduction to MongoDB 

MongoDB is a NoSQL, document-oriented database that uses JSON like documents with optional schemas. 

Some of the common features of MongoDB are as follows - 

- Document Model - MongoDB stores data in flexible, JSON like documents.

- It is designed to handle a large volume of reads and writes, providing high performance for data-intensive applications.

- Schema-less - It Supports dynamic schemas, meaning you can create documents without defining the structure in advance. 

- Scalability - It supports scaling through a process called sharding, where data is distributed across multiple servers. 

Some basic concept that one should be familiar with are as follows - 

- A MongoDB database holds collections of documents. Each database is identified by a name. 

- Documents are the basic unit of data in MongoDB and are represented in BSON(Binary JSON) format. It is essentially a set of key-value pairs. 

- Replica Set - It is a group of MongoDB servers that maintain the same data set, providing redundancy and high availability. 

- Sharding - It is a method for distributing data across multiple servers, allowing MongoDB to handle large data sets.

### Types of Databases 

1. Relational Databases

    - It stores the data in tables with rows and columns.

    - Uses SQL for defining and manipulating the data.

    - Example - MYSQL, SQL Server.

2. NOSQL Databases

    - These are designed for the purpose of handling large-scale data storage and for applications that require real-time processing.

    Some of the different types of NOSQL databases are as follows - 

    - Document-Oriented - It stores the data in documents similar to JSON.

    - Key-Value - It stores the data as a collection of key-value pairs. 


3. Object-Oriented Databases 

    - It stores the data as objects, similar to object-oriented programming languages.

    - Example - ObjectDB
