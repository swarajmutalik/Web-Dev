# Day 50 Overview 

## Advanced JS Concepts 

1. IIFE 

- It is a JS function that is executed as soon as it is defined. 

- The typical syntax mainly involves wrapping a function in parentheses and immediately invoking it. 

- It is useful for intializing code that should run once and not be accessible later. 

- IIFE are often used to encapsulate the code, often helping in preventing variable name conventions and keeping the global scope clean. 

2. Destructuring 

- It is a JS feature that allows you to unpack values from arrays or  properties from an object into distinct values. 

3. Spread Operator 

- The spread operator in JS allows an interable to be expanded in places where zero or more arguments or elements are expected. 

4. Hoisting 

- It is a JS mechanism where variables and function declarations are moved to the top of their contianing scope during the compile phase before the code is executed.

### Global, Local and Block Scope 

1. Global Scope 

- Variables declared inside the global scope are accessible from anywhere in the code. 

- They are defined outside of any function or block.

2. Local Scope 

- It is also known as function scope and in this the variables are declared within a function.

- They are local to the function and it cannot be accessed from outside. 

3. Block Scope 

- Variables declared with 'let' or 'const' within a block are block-scoped. 

- They are only accessible within that block.
