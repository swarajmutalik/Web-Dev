# Javascript Callbacks and Promises

- A Callback in JS is a function that is passed as an argument to another function and is executed later.

- The main issue with the callbacks is the Callback Hell i.e., when multiple nested callbacks are used, it leads to callback hell, making it difficult to read and maintain the code.

## Promises in JS

- It is an object representing the eventual completion or failure of an async operation.

- A Promise has the following states -

  - Pending - initial state, before completion or failure.

  - Resolved - Operation completed successfully.

  - Rejected - Operation failed.

- Promises can be chained to avoid callback hell and improve the readability.
