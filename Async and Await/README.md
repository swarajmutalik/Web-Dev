# Async/Await and Fetch API

- JS provides a powerful way to handle asynchronous operations using `Async/Await` and the `Fetch API`.

- These features make working with promises easier and more readable compared to the normal callback-based approaches.

1. `Fetch API`

    - The Fetch API is used to make HTTP requests and handle responses.

    - It returns a Promise, which resolves once the reqeust is completed.

2. `Async/Await`

    - It is a modern way to handle async code, making it look more like synchronous code.

    - An async function always returns a promise, allowing you to use await inside it.

## Advantages of Async/Await Over Promises

- `Readability` - Code is easier to understand.

- `Avoids Callback Hell` - There is no need of deeply nested callbacks.

- `Error Handling` - Try/catch makes handling errors more intuitive.
